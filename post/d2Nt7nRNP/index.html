SolrCloud-tlog、recovery

2018-09-08

本文参考：Understanding Transaction Logs, Soft Commit and Commit in SolrCloud

recovery过程中会存在很多坑，如果不知道原因，会很头疼，例如recovery了一会之后，从节点变为down了，是不是很懵逼，其实呢在recovery过程中，如果leader发现与从节点通信失败，则会主动将从节点的状态置为down，而且从节点在recovery进程中还无法直接将一个down状态转换为active，只能由leader发送leader initial recovery请求触发从节点再次recovery，直到状态OK。

一上来就说这，貌似有点枯燥，当然啦，说这些呢也是为了督促我好好了解recovery。

Recovery策略

Solr中recovery有两种策略，这两中策略也象征了recovery的程度：

  • peer sync
  • snapshot replication(replicating)

何时启用peer sync何时启用replicating，取决于从与主上版本的差异大小，如果从发现与主上的索引已经版本相差小于100，则使用peer sync，否则使用全量同步。
